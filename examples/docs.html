Pigeon HTTP API

Governed messaging over HTTP. The API is versioned by path - every resource lives under /v1; a future breaking change ships as /v2 so existing clients keep working. Every call carries the caller's identity and region as headers, and the broker enforces the subject's policy on each publish and receive.

Base URL
-
Identity headers
x-pigeon-principal: spiffe://.../sa/your-service
x-pigeon-region:    uk
Error shape
{
  "error": {
    "code": "POLICY_DENIED",
    "message": "...",
    "details": {}
  }
}

Codes map to HTTP status: 403 policy/ region/intent, 422 validation/sensitive-field, 400 malformed, 404 unknown subject, 413 too large, 405 wrong method.

Served by the running Pigeon broker. Source: github.com/vishnu-77/pigeon. Try-it calls hit this origin and are governed for real.
